# 长篇小说协作项目

## 项目概述
现代玄幻长篇小说。docs/ 存放设定和规划文档，pages/ 存放正文。

### 结构规模
- 全书约 **10 章**（章 = 大篇章/卷/arc）
- 每章约 **100-200 节**
- 每节约 **2000 字**
- 总计约 **1000-2000 节**

### 术语约定
- **章**：大篇章，对应一段完整的剧情弧（如"第一章：觉醒"）
- **节**：章内的最小叙事单元，每节一个文件，约 2000 字

## 写作工作流（每次写新节必须遵守）

### 写之前
1. 读 `docs/outline.md` 了解总体方向
2. 读当前章的大纲（如 `docs/chapter01_outline.md`）了解本节在剧情中的位置
3. 读 `docs/characters.md` 确认角色状态
4. 读 `docs/continuity.md` 确认已建立的事实，避免矛盾
5. 读前两节正文（`pages/` 中最近的两个文件），衔接文风和剧情

### 写的时候
- 每节约 2000 字
- 文件命名：`pages/ch01/001.md`（章文件夹 + 三位数节编号）
- 节开头标注：`# 第一章·第一节：节名`
- 保持林时的第一人称内心独白风格：自嘲、幽默、偶尔真情流露
- 台词要口语化，像真人说话
- 不刻意解释世界观，让读者通过事件自然了解
- 战斗场面重感受轻招式，不写"他使出了XX技能"这种
- 情感描写克制，少用形容词堆砌，靠细节和动作传递

### 写完之后
1. 更新 `docs/continuity.md`——记录本节新建立的事实、伏笔、角色状态变化
2. 如果涉及新角色或角色重大变化，更新 `docs/characters.md`
3. 如果剧情走向与大纲有调整，更新对应的章大纲文件
4. 每写完一节，在 continuity.md 的章节摘要表中追加一行

### 连续写作模式
当用户要求连续写多节时（如"写5节"），按以下流程循环：
1. 读必要的上下文文件（首节全读，后续只读上一节正文 + continuity.md）
2. 写正文
3. 更新 continuity.md
4. 进入下一节，重复

## docs/ 文件说明
- `outline.md` — 总大纲，世界观、结构、基调
- `characters.md` — 角色设定圣经
- `continuity.md` — 连续性追踪（事实、伏笔、角色状态、节摘要）
- `chapter{NN}_outline.md` — 各章详细剧情大纲
- `timeline.md` — 故事内时间线

## 风格参考
- 龙族（江南）：自嘲式幽默、压抑底色、少年感
- 基调：压抑中带稀少的温暖
- 禁止：中二病台词、过度热血、大段说教、无意义的日常水字数
